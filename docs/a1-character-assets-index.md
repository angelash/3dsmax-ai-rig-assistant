## 参考骨架观察

- 28 个 JSON 均显示 `Generation = AccuRig`，版本字段为 `1.10.1822.1`。
- 主流参考骨架包含 72 个 `Physics > Collision Shapes` 条目，骨名以 `RL_BoneRoot`、`CC_Base_*` 为主。
- `甘宁`、`关羽`、`吕布` 的 JSON 为 70 shapes，相比 72-shape 模板少 `RL_BoneRoot` 和 `_026`。这三例不要直接当作“错误”，先作为导出差异/工程文件差异样本记录。
- `小兵` JSON 文件名是 `伏兵模型.json`，但参考绑定 FBX 是 `伏兵绑定骨骼模型.fbx`，自动匹配时需要显式映射。
- `甄姬` JSON 与 `甄姬模型.fbx` 同名，说明这个 FBX 很可能已经是参考绑定版本；使用前必须打开确认骨架和 Skin。

## 校对与学习用法

1. 选一个完整对照样本，把 `<角色>模型.fbx` 作为输入运行 Stage01/自动绑骨流程，保留参考绑定 FBX/JSON 不改动。
2. 生成候选骨架后，先做机械校对：骨名集合、根骨/骨盆/脊椎/头/锁骨/肩肘腕/髋膝踝/脚趾、左右镜像关系、武器挂点是否覆盖参考答案的关键结构。
3. 再做空间校对：把候选骨点和参考绑定骨点导出为同一坐标系，比较关节位置、骨段长度、左右对称误差、头身比例、手脚 pivot、裙摆/披风/靴筒遮挡下的隐藏腿轴。
4. 进入 Skin 前必须走多视图语义签核：正/侧/顶包裹性、头颈、骨盆、手脚、武器、服饰遮挡、体块中心链都要通过；不能只看数值评分。
5. Skin/权重校对建议从参考绑定中提取影响骨列表、每顶点影响数量、刚性道具权重、左右权重镜像、测试动作变形。教程里的默认经验是卡通手游角色先用较少影响数做清晰权重，再按变形需要加细节。
6. 把失败案例按类型沉淀：骨盆偏高/偏低、肩宽追衣服外轮廓、膝盖追裙摆、脚掌 pivot 不在鞋底、头饰拉高 HeadTop、武器未刚性跟随、手部团块误判为手指。

## 推荐优先级

- 快速回归：`左慈`、`曹操`、`大乔`、`张飞`。文件体量较小，结构完整，适合频繁跑。
- 压力样本：`甘宁`、`关羽`、`吕布`。绑定 FBX 较大，JSON 为 70 shapes，还带工程/profile 文件，适合检查导出差异。
- 武器/道具样本：`貂蝉`、`关羽`、`孙尚香`、`小兵`。武器体量或道具数量更复杂，适合训练挂点和刚性道具规则。
- 盲测样本：`袁绍`、`赵云`。目前只有原模型，适合在没有参考答案时验证泛化能力；等补参考绑定后再反算差异。
